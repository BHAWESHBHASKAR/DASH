# Reference

The reference section is the **wire-level** description of DASH. It is precise, exhaustive, and not the place to start if you are new to the project — see the [Guides](../guides/index.md) for task-oriented documentation.

## Pages

- [**HTTP API**](api.md) — every endpoint, every method, every request and response shape, every error code.
- [**Configuration**](configuration.md) — every environment variable, the default, and the meaning.
- [**Benchmarks**](benchmarks.md) — the methodology and the latest published numbers.
- [**Architecture decisions**](architecture-decisions.md) — the ADRs that explain *why* DASH is shaped the way it is.

## How to read this section

Reference pages are written to be searchable and skim-readable. If you are looking up a specific field or a specific error code, jump to the page and Ctrl-F. If you are reviewing a design decision, read the [ADRs](architecture-decisions.md).
